  1. Polar Media Consensus Graph:

    This utility periodically computes the similarity of news stories (nodes) from left/center/right news media, and draws an edge between highly similar stories. The graph can be used to track the development of a news event, identify exclusive consensus within left, center, or right media, or consensus across all (left-right). The method can also be used to track a story over time.

    2. User-focused activity for a URL:

      This utility can be used to identify the major "players" in spreading a story, and potentially identify bot activity.

      Visualizes the network of users who shared a link on Twitter. The proportion of the nodes represent the magnitude of the share - users (nodes) who tweeted a lot about the link have larger nodes than users who tweeted a little about the link. If a user A, retweets a user B, you would see an arrow from A to B (inlink to B). A user with a tweet that has a high volume of retweets is represented by a node with a high proportion of inlinks. A user who retweets at a high volume, is represented by a node with a high proportion of outlinks. Colors represent the various sources of the tweet (e.g, Twitter web client or an automatic tweeting system like ift.tt).

    3. Quantifying consensus of links using tweets:

      This utility can be used to identify links that are used in similar contexts. In other words the utility identifies links that are similar based on how they are used by users on Twitter. Hashtags encode the use intent or association of a tweet author to a link. Consequently, similar links may co-occur with a common set of hashtags and vice versa.

      This utility quantifies the similarity of stories (links) embedded in tweets indirectly by calculating the similarity of the hashtags that co-occur with the links. The higher (~1.0) the similarity between two links, the higher the blue opacity of the cell that represents the similarity score of the link pair, and vice versa. For example, the cell representing similarity score of 1.0 (100% in common) of a link pair has a blue opacity, but the cell representing the similarity score of 0 (nothing in common) of a link pair is represented by a white color.

    4. (Implementation pending) Activities for users and stories:

      This utility is very similar to 2.1, but differs as follows: 2.1 focuses on the activity for a single story by exploring the users (nodes) who shared the story and the interactions (edges) between the users. However, this utility focuses on multiple stories unlike 2.1.

      This utility explores the various user activities across multiple stories. It can be used to identify the most actively shared stories and the interaction dynamics (retweets & tweets) of users who shared them. There are two kinds of nodes, namely, user nodes and story nodes. Consider the following:

      - An edge from a user node to story node indicates that the user shared the story (arrow from the user to the story)
      - An edge from a user node A to a user node B indicates that user A retweeted user B for a given story.
      - The size of a user node is proportional to the amount of retweets a user received.
      - The size of a story node is proportional to the amount of users who shared the story.

    5. (Implementation pending) Similarity of tweet texts:

      This utility can be used to identify highly similar/duplicate tweets for a story. Some bot or coordinated messaging activities can be identified by a high degree of similarity in tweet texts.

      This matrix shows a way of identifying high similarity (dense collection of blue). Given a pair of tweets, a similarity value of 1 (100% in common) is represented by a blue cell, and a similarity of 0 (nothing in common) is represented by a white cell. Large clusters of blue indicate highly similar tweets.
